运算符重载

发表于 2020-08-17 更新于 2020-08-20 阅读次数：
本文字数： 11k 阅读时长 ≈ 10 分钟

基本概念说明

  • C++预定义了运算符表示对数据的运算：+, -, *, /, %, ^, &, ~, !, |, =, <<, >>, != ；
    • 但这些只能用于基本的数据类型：整型, 实型, 字符型, 逻辑型。
  • 要想运用于自定义数据类型，就得进行运算符重载
    • 对抽象数据类型也能够直接使用C++提供的运算符。
    • 使得程序更简洁；使得代码更容易理解。
  • 运算符重载：
    • 对已有的运算符赋予多重的含义。
    • 使同一运算符作用于不同类型的数据时，有不同类型的行为。
    • 目的：扩展C++中提供的运算符的适用范围, 以用于类所表示的抽象数据类型。
    • 同一个运算符, 对不同类型的操作数, 所发生的行为不同。
      • 如：(5,10i) + (4,8i) = (9,18i)表示复数运算；5 + 4 = 9表示正数运算。
  • 运算符重载的实质是函数重载，可以用如下写法
  • 运算符重载时，程序编译过程大概为：
    • 把含运算符的表达式对应转换成为：对 运算符函数的调用。
    • 把运算符的操作数（a+b中的a、b）作为运算符函数的参数，来进行传递。
    • 运算符被多次重载时, 根据 实参的类型 决定调用哪个运算符函数。
  • 运算符可以被重载成普通函数。
  • 也可以被重载成类的成员函数。
  • 运算符重载为普通函数：
    • 重载为普通函数时, 参数个数为运算符目数
  • 运算符重载为成员函数
    • 重载为成员函数时, 参数个数为运算符目数减一

赋值运算符“=”的重载

  • 进行重载后，赋值运算符 两边的类型 可以 不匹配

    • 如：把一个 int类型变量 赋值给一个 Complex对象；
    • 把一个 char * 类型的字符串 赋值给一个 字符串对象。

  • 注意：赋值运算符 “=” 只能重载为成员函数；

  • 举例：编写一个长度可变的字符串类String
    • 包含一个char * 类型的成员变量，用于指向动态分配的存储空间；
    • 该存储空间用于存放 ‘\0’ 字符串结束符结尾的字符串。

  • 重载赋值运算符的意义—用于明确浅复制和深复制。

    • 如： s1 = s2 ;在没有相应的重载赋值运算符时，编译器会自动调用 浅复制/浅拷贝。

对重载运算符后，返回值类型的讨论?!

  • 对 operator = 返回值类型的讨论：

    • 使用 void 作为返回值类型好不好？
    • 考虑：a=b=c;//等价于 a.operator=(b.operator=(c));
    • 此时发现，若返回值是 void 会报错的。
    • 因此返回值不能是 void 。必须得是 String 或 String & 类型。

  • 那直接使用 String类型 好不好，不使用 String &？

    • 可以，但是不好；
    • 因为运算符重载时, 好的风格 — 尽量保留运算符原本的特性
    • 如： (a=b)=c; //最终目的是会修改a的值
    • 即等价于：(a.operator=(b)).operator=(c);
    • 若返回值类型为 String 则修改的不是a的值，而是新生成的 String 对象。
    • 因此 应返回 赋值运算符左边对象的引用，即 String&。

  • 上述的 String 类是否没有问题了？！不是，还有问题，永远不要忘记考虑“复制构造函数”和“类型转换构造函数”。

    • 为 String类编写 复制构造函数 时，会面临和 ‘=’ 同样的问题, 用同样的方法处理。（即浅复制的问题，内存垃圾和重复释放内存空间）
    • 类型转换构造函数，则因使用情况而定。

将运算符重载为友元函数

  • 通常, 将运算符重载为类的成员函数
  • 重载为友元函数的情况，是因为：
    • 成员函数不能满足使用要求；
    • 普通函数, 又不能访问类的私有成员。
  • 为了使得上述表达式能成立, 需要将+重载为普通函数（传入+号两边的表达式作为函数参数）。
  • 由于普通函数不能类的访问私有成员，因此要声明为 友元函数。

流插入(<<)运算符和流提取(>>)运算符的重载

  • 问题提出：cout << 5 << “this”;为什么能够成立？cout是什么？“<<” 为什么能用在 cout上？
  • cout 是在 iostream 中定义的，ostream 类 的对象。

  • “<<” 能用在 cout 上是因为，在 iostream 里对 “<<” 进行了重载。

  • 考虑，怎么重载才能使得 cout << 5; 和 cout << “this”都能成立 ？

  • 流插入运算符(<<)的重载:

    • 使得满足：cout << 5;即 cout.operator<<(5);
    • cout << “this”; 即 cout.operator<<( “this” );
    • cout << 5 << “this” ; 即：cout.operator<<(5).operator<<(“this”);
  • 实例:下述程序输出 5hello,该补充些什么
  • 举例：复数的输入输出
    • 假定c是Complex复数类的对象，现在希望 写“cout << c;”，就能以“a+bi”的形式输出c的值，写“cin>>c;”，就能从键盘接受“a+bi”形式的输入，并且使得 c.real = a,c.imag = b。

自增/自减运算符的重载

  • 自增 ++/自减 — 运算符有 前置/后置 之分。

  • 前置运算符作为一元运算符重载

    • 重载为成员函数:
      • T & operator++();
      • T & operator--();
    • 重载为全局函数：
      • T & operator++(T &);
      • T & operator—(T &);
    • ++obj, obj.operator++(), operator++(obj) 都调用上述函数

  • 后置运算符作为二元运算符重载

    • 多写一个参数, 但具体无意义
    • 重载为成员函数:
      • T operator++(int);
      • T operator--(int);
    • 重载为全局函数:
      • T operator++(T &, int);
      • T operator--(T &, int);
    • obj++, obj.operator++(0), operator++(obj,0) 都调用上函数
  • 举例：设计 CDemo 类实现上述功能

运算符重载的注意事项

  1. C++不允许定义新的运算符，只能重载之前的运算；
  2. 重载后运算符的含义应该符合日常习惯，看起来很合理，符合基本的数学运算等。遵循好的风格：尽量保留运算符原本的特性。
    • complex_a + complex_b //复数相加
    • word_a > word_b //单词比较大小
    • date_b = date_a + n //日期增加
  3. 运算符重载不改变运算符的优先级；
  4. 以下运算符不能被重载: “.”, “.*”, “::”, “?:”, sizeof
  5. 重载运算符(), [], ->或者赋值运算符=时, 重载函数必须声明 为类的成员函数
