函数模板与类模板

函数模板

  • 在理解模板之前，首先引进 泛型程序设计（Generic Programming）的概念：
    • 算法实现时不指定具体要操作的数据的类型，在实际使用算法时再给出具体的数据类型；
    • 泛型：算法实现一遍，可适用于多种数据结构；
    • 优势: 减少重复代码的编写；
    • C++ 中实现泛型程序设计的工具，就是模板；
    • 大量编写模板, 使用基于模板的程序设计可以分为两类：函数模板 和 类模板。

  • 举例：如何只写一个Swap函数（交换a,b的值）， 就能交换各种类型的变量?可用函数模板解决。

函数模板的几点注意

  • C++编译器匹配函数实现的顺序：
    1. 先找参数完全匹配的普通函数(非由模板实例化 而得的函数)；
    2. 再找参数完全匹配的模板函数；
    3. 再找实参经过自动类型转换后能够匹配的普通函数；
    4. 上面的都找不到，则报错。
  • 要注意：赋值兼容原则引起函数模板中类型参数的二义性

类模板

为什么需要类模板？

  • 对于上述的数组类，除了元素的类型不同之外, 其他的完全相同。因此可以使用类模板。
  • 类模板：在定义类的时候给它一个/多个参数，用这个/些参数表示不同的数据类型，来定义类模板。
  • 在调用类模板时, 指定参数, 由编译系统根据参数提供 的数据类型自动产生相应的模板类。

类模板与非类型参数

  • 类模板的参数声明中可以包括非类型参数：**template < class T, int elementsNumber>**
  • 非类型参数: 用来说明类模板中的属性；
  • 类型参数: 用来说明类模板中的属性类型, 成员操作的参数类型和返回值类型。
  • 但要注意：CArray<int,40 >和CArray<int,50 >完全是两个类 这两个类的对象之间不能互相赋值.
